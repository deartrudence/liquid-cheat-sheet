Liquid is an open-source template language created by Shopify and written in Ruby. It is the backbone of Shopify themes and is used to load dynamic content on storefronts.

Template Variables ‚ñ∫

Shopify objects that contain attributes that can be dynamically put on the page. For example, the product object contains an attribute called title that can be used to output the title of a product.

The handle is used to access the attributes of a Liquid object. By default, it is the object‚Äôs title in lowercase with any spaces and special characters replaced by hyphens (-). Every object in Liquid (product, collection, blog, menu) has a handle.

Liquid Tags ‚ñ∫

There are two types of tags; Output Statment Tags are used to display information on the page and uses {{ }} and Logic statment Tags are used to perform logic and uses {% %}

The following tags are logic based and therefore normally use {% %}

Variable Tags ‚ñ∫

assign ‚ñ∫

Creates a new variable


{% assign var-name = 'elephant' %}
{{ var-name }}

returns

elephant
capture ‚ñ∫

Captures the string inside of the opening and closing tags and assigns it to a variable. Variables created through {% capture %} are strings. This is very useful for capturing information that's being looped over.


{% capture item_list %}
  {% for item in cart.items %}
      {{ item.product.title }}
  {% endfor %}
{% endcapture %}

returns

item1 item2 item3

assuming those are the items in your cart.

Control Flow Tags ‚ñ∫

if ‚ñ∫

Acts as a standard if statement and executes a block of code only if a certain condition is met


{% if product.title == 'the blue book' %}
  You are buying the blue book!
{% endif %}
unless ‚ñ∫

Like if, but executes a block of code only if a certain condition is not met (ie. result is false).


{% unless product.title == 'the blue book' %}
  You are NOT buying the blue book!
{% endunless %}
case ‚ñ∫

Creates a switch statement to execute a particular block of code when a variable has a specified value. case initializes the switch statement, and when statements define the various conditions. The else statement is used as a catchall to execute if none of the conditions are met.


{% case product.title %}
  {% when 'The blue book' %}
     You're getting the blue book
  {% when 'The red book' %}
    You're getting the red book
  {% else %}
     You're getting another color book
{% endcase %}

This tag allows you to put comments in your liquid code.

include ‚ñ∫

Inserts a snippet from the snippets folder of a theme.


{% include 'snippet-name' %}

You can also include variables to be used within the snippet.


{% include 'snippet', first_var: 'books', second_var: 'videos' %}

The with parameter assigns a value to a variable inside a snippet that shares the same name as the snippet. If snippet is named 'fruit.liquid' you can use:


{% include 'fruit' with 'peach' %}

Pagination ‚ñ∫

The paginate tag is built using the attributes of the paginate object. You can use the default_pagination filter as a quick and easy alternative.


{% assign collection = collections.frontpage %}
{% paginate collection.products by 8 %}
	

	  
Collection: {{ collection.title }}

	  {% for product in collection.products %}
	    {% include 'product-loop' %}
	  {% endfor %}
	



{% if paginate.pages > 1 %}


  
{{ paginate | default_pagination }}



{% endif %}
{% endpaginate %}

This code shows the variable collection being assigned to collections.frontpage. Then we use the paginate tag to start and declare that we want to paginate through the collection.products with 8 per page. Below this is where we display the products using the product loop snippet. Once that is done, we use the paginate object to check to see if there is more than one page. If so, we use the paginate filter to set default pagination navigation. We then end the whole section with the {% endpaginate %} tag

Liquid Tag ‚ñ∫

The paginate tag works with thefor tag to split content into many pages. It must wrap a for tag block that loops through an array, as shown in the example below:


{% paginate collection.products by 8 %}
  {% for product in collection.products %}
    
  {% endfor %}
{% endpaginate %}
